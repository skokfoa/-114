# Program R 最小路徑

此專案是一個使用 Python 撰寫的迷宮最短路徑求解器，目標是計算從迷宮起點到終點所需的最小移動成本。

## 功能介紹
- 使用 **動態規劃 (Dynamic Programming)** 與 **記憶化搜索 (Memoization)** 計算最小花費。
- 支援動態輸入迷宮尺寸與權重陣列。

## 如何使用
1. 執行程式：`python Program_R.py`
2. 輸入迷宮維度 `n` (行) 和 `m` (列)。
3. 依序輸入迷宮的權重數值。

## 演算法說明
- 程式透過遞迴方式向下與向右探索，並使用 `min_cost` 二維陣列記錄目前到達各點的最小成本以避免重複計算。這種 Top-down 的動態規劃作法可以將時間複雜度降至 $O(n \times m)$，大幅提升執行效率。

# Program S 最小生成樹

此專案旨在解決「最小生成樹 (Minimum Spanning Tree, MST)」問題，透過優化道路連接方式來計算最多可以省下多少成本。

## 功能介紹
- **無結構化 (Class-Free) 實作**：捨棄自訂的 Class 結構，直接利用 `(cost, dot1, dot2)` 這樣的 tuple 來取代原本的結構。這讓 Python 在排序時能自動比較成本，使程式碼更精簡且執行速度更快。
- 實作 **Kruskal 演算法**：挑選全域最輕的邊 (配合 Union-Find)。適合稀疏圖 (Sparse Graph)。
- 實作 **Prim 演算法**：從已連接的節點向外拓展最輕的邊 (配合 Priority Queue)。適合密集圖 (Dense Graph)。
- 支援多筆測資 (Test Cases) 處理。

## 使用方式
1. 執行程式：`python Program_S.py`
2. 輸入測試資料筆數 `T`。
3. 依照提示輸入路口數量 `m` 與道路數量 `n`。
4. 輸入所有道路的連接路口 `x`, `y` 以及權重 `cost`。
5. 根據提示選擇要使用的演算法 (1: Kruskal, 2: Prim)。
6. 程式將輸出優化後所省下的總成本。

## 演算法邏輯與複雜度
- **Kruskal**：先對邊依權重從小到大排序，利用 Union-Find 檢查連通性避免形成環。時間複雜度為 $O(E \log E)$。
- **Prim**：從任意起點出發，每次利用優先佇列挑選連接已訪問區域的最小邊。時間複雜度為 $O(E \log V)$。
